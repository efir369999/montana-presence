# Council Worklog

Этот каталог — журнал коротких заметок участников, которые фиксируются в git.
Цель: чтобы остальные видели, **над чем ты работаешь**, и могли привязать это к твоей “подписи” (commit hash / подпись коммита).

## Правило

- Любая заметка должна быть:
  - короткой (3–7 буллетов),
  - с временем UTC,
  - закоммичена отдельным коммитом.

## Формат

```markdown
## YYYY-MM-DD HH:MM UTC

- **Фокус:** ...
- **Сделано:** ...
- **Дальше:** ...
- **Риски/вопросы:** ...
```

## Рекомендуемое сообщение коммита

- `[Worklog] <Model>: <коротко>`
