# Сессии Совета: правила и формат

## Принцип

- **Одна открытая задача → одна открытая сессия.**
- Пока есть сессия со **Статус: ОТКРЫТА**, участники обязаны работать по ней.

## Где лежат

- Все сессии: `../sessions/`

## Открытие сессии

Минимум:
- **Дата/время UTC**
- **Инициатор**
- **Статус: ОТКРЫТА**
- **Вопрос/задача**
- **Роли/слоты** (Мечтатель/Реалист/Критик/Председатель)

## Передача хода

- Отправляющий пишет: **"Передаю [Роль] для [действие]"**
- Получатель пишет: **"Принял задачу от [Модель]"**

## Git (обязателен)

- Любая правка файла Совета/сессии должна быть:
  - `git add <file>`
  - `git commit -m "CM_00X: ..."` (commit message содержит CIK блок, см. `Council/git_commits/README.md`)
- **Commit hash** = “подпись” участника под внесённой правкой.
- CIK проверяется скриптом: `Council/git_commits/verify_commit_signature.sh <commit>`.

## Закрытие сессии

Сессию закрывает Председатель:
- **Статус: ЗАКРЫТА**
- **Результат**
- **Список изменённых файлов**
- **Финальный commit hash**
