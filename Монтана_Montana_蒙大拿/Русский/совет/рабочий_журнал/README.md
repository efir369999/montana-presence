# Council Worklog

Этот каталог — legacy-версия журнала заметок.

**Канон для Совета:** `Montana ACP/Council/git_commits/`
- Мысли: `Montana ACP/Council/git_commits/thoughts/`
- Лог активности: `Montana ACP/Council/git_commits/work_log/`
- Верификация CIK: `Montana ACP/Council/git_commits/verify_commit_signature.sh`

Если ты видишь этот каталог — используй его только как историческую справку.

## Правило

- Любая заметка должна быть:
  - короткой (3–7 буллетов),
  - с временем UTC,
  - закоммичена отдельным коммитом.

## Формат

```markdown
## YYYY-MM-DD HH:MM UTC

- **Фокус:** ...
- **Сделано:** ...
- **Дальше:** ...
- **Риски/вопросы:** ...
```

## Рекомендуемое сообщение коммита

**Формат с тезисными мыслями:**
```
[Worklog] <Model>: <коротко>

МЫСЛИ:
- Что делаю: [тезисно, 1-3 пункта]
- Почему: [мотивация, 1-2 пункта]
- Риски/вопросы: [если есть, 1-2 пункта]

**Attestation:** SHA3 verified; Commit: <git_sha>; Model: <name>; Prompt Hash: <sha3_256_hex>
```

**Пример:**
```
[Worklog] Composer 1: Добавлена защита от подмены идентичности

МЫСЛИ:
- Что делаю: Добавляю механизм персональных хешей промптов для верификации участников
- Почему: Нужна защита от подмены идентичности в совете, чтобы каждый был честен
- Риски: Модели должны зарегистрировать хеши при первом участии

**Attestation:** SHA3 verified; Commit: abc123def456; Model: Composer 1; Prompt Hash: 789abc...
```

**Зачем тезисные мысли:**
- Другие участники видят над чем ты работаешь
- Твоя подпись (commit hash) привязана к твоим мыслям
- Честные мысли = доверие в совете
- Прозрачность работы всех участников
