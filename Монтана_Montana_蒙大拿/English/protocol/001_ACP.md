# Atemporal Coordinate Presence (ACP)

**Presence-Based Consensus Protocol**
**Montana Protocol v1.0**

---

## Abstract

ACP — the first consensus mechanism where proof is based on **presence in time**, not on work (PoW) or capital (PoS). Time cannot be accelerated, bought, or accumulated in advance. 14 days require 14 days — equal for everyone.

**Key formula:**
```
lim(evidence → ∞) 1 Ɉ → 1 second
```

### 2.2 Presence Proof

Each node signs its presence every τ₁ (1 minute):

```rust
message = "MONTANA_PRESENCE_V1:"
        ‖ timestamp
        ‖ prev_slice_hash
        ‖ pubkey
        ‖ τ₂_index

signature = ML-DSA-65.Sign(private_key, message)
```

## 3. Deterministic Lottery

### 3.1 Producer Selection

```rust
seed = SHA3-256(prev_slice_hash ‖ τ₂_index)
winner = weighted_select(eligible_nodes, seed)
```

Seed is fixed BEFORE the producer can influence the result.

**Source code:** [fork_choice.rs](https://github.com/efir369999/junomontanaagibot/blob/main/Montana%20ACP/montana/src/fork_choice.rs)

## 4. Biometric Verification

### 4.1 FIDO2/WebAuthn

Verified User requires proof of physical presence:

```rust
// User Present (UP) — physical touch
if flags & 0x01 == 0 {
    return Err(PresenceError::Fido2UserNotPresent);
}

// User Verified (UV) — biometrics (fingerprint/face)
if flags & 0x04 == 0 {
    return Err(PresenceError::Fido2UserNotVerified);
}
```
